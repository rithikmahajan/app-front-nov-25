## Key Takeaways

1. **Backend is notified BEFORE local state is cleared**
   - Ensures backend gets logout notification with valid token
   - Even if backend call fails, local logout succeeds

2. **Automatic state synchronization**
   - Detects mismatches between Firebase and backend auth
   - Automatically syncs states on app start

3. **Graceful degradation**
   - Local logout always succeeds
   - Backend notification is best-effort
   - User experience is never blocked

4. **Clear logging**
   - Every step is logged
   - Easy to diagnose issues
   - Can verify correct behavior in logs

5. **Guest session ready**
   - After logout, user can browse as guest
   - No interruption in app functionality
   - Can sign in again anytime
